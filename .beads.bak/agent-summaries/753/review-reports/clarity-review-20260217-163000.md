### Finding 5: reviews.md Step 0 ls command uses glob patterns that may match wrong timestamps

- **File(s)**: `/Users/correy/projects/ant-farm/orchestration/templates/reviews.md`:341-344
- **Severity**: P3
- **Category**: clarity
- **Description**: The Step 0 verification command uses `ls .beads/agent-summaries/<epic-id>/review-reports/clarity-review-*.md` with a wildcard. If previous review cycles left reports in the same directory (from earlier timestamps), the glob would match all of them and the existence check would pass even if the current cycle's reports are incomplete. The instructions say "verify all 4 expected files exist" but the glob pattern does not filter by the current timestamp.
- **Suggested fix**: Use the specific timestamp in the ls command: `ls .beads/agent-summaries/<epic-id>/review-reports/clarity-review-<timestamp>.md` etc. The timestamp is available from the Queen's input.
- **Cross-reference**: Potential edge case -- could produce false positive on existence check if stale reports exist.

### Finding 6: reviews.md Correctness Redux instructs running bd show, but agent is a Nitpicker

- **File(s)**: `/Users/correy/projects/ant-farm/orchestration/templates/reviews.md`:188-189
- **Severity**: P3
- **Category**: clarity
- **Description**: Lines 188-189 tell the Correctness Redux reviewer to "Run `bd show <task-id>` for each task in the commit range to retrieve the original acceptance criteria." Nitpickers are spawned as `code-reviewer` agent types. The code-reviewer agent definition may or may not have Bash tool access to run `bd show`. Even if it does, this instruction is the only review type that asks the reviewer to run bd commands, making it inconsistent with the other three review types. The instruction also says "Do not rely solely on the orchestrator's prompt" which implies the data file might be incomplete -- a trust issue worth surfacing.
- **Suggested fix**: Either ensure the Correctness Redux data file includes the full acceptance criteria (eliminating the need for `bd show`), or add Bash to the code-reviewer tool list and note the exception in the review setup.
