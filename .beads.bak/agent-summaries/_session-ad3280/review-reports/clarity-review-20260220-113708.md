### Finding 8
- **File**: `/Users/correy/projects/ant-farm/scripts/compose-review-skeletons.sh:99-102`
- **Severity**: P3
- **Category**: Comment accuracy
- **Description**: The comment on line 99 says "First convert ALL single-brace uppercase placeholders to double-brace format: {WORD} -> {{WORD}}". The sed pattern on line 102 is `s/{\([A-Z][A-Z_]*\)}/{{\1}}/g`. This regex requires the first character to be uppercase (`[A-Z]`) and subsequent characters to be uppercase or underscore (`[A-Z_]*`). The comment says "WORD is all-caps with underscores" which is accurate, but does not mention that the regex also requires at least two characters (one `[A-Z]` plus at least one `[A-Z_]`). A single-character placeholder like `{X}` would not match. This is likely intentional (no single-char placeholders exist) but the comment does not say so.
- **Suggested fix**: Clarify the comment: "where WORD is 2+ uppercase characters/underscores, e.g. {REVIEW_TYPE}". Or adjust the regex to `[A-Z_]+` if single-char should match too.

### Finding 9
- **File**: `/Users/correy/projects/ant-farm/scripts/fill-review-slots.sh:236-238`
- **Severity**: P3
- **Category**: Code clarity
- **Description**: The expected_paths construction on lines 234-238 builds a newline-separated list with `\n` in a string, then uses `printf '%b'` piped to `sed '$d'` to remove the trailing newline. The `sed '$d'` command deletes the last *line* from the output, not just a trailing newline character. If the list has entries, this removes the last entry entirely rather than a trailing blank line. This appears to work because each entry ends with `\n`, creating an extra blank line, and `sed '$d'` removes that blank line. But the intent ("Remove trailing newline") does not match the mechanism ("Delete last line"), making this fragile and confusing.
- **Suggested fix**: Replace with a clearer approach: build the list without a trailing `\n`, or use `printf '%b' "$expected_paths" | sed '/^$/d'` to remove blank lines instead of the last line.
