## Selected Approach with Rationale

**Approach 5: Structured Conflict Metadata**

The consolidation report now includes a dedicated "Severity Conflicts" section triggered when any two reviewers assign severities to the same root cause that differ by 2+ levels (e.g., P1 vs P3, P2 vs P4).

For each conflict, the section includes:
1. Root cause title
2. Disagreeing severity assignments (e.g., "Reviewer A: P1, Reviewer B: P3")
3. Which reviewers were involved
4. Brief explanation of why the assessment may differ (e.g., different review domains)
5. The final severity used (highest) for the issue
6. Explicit flag that Queen should review before issue closure

This approach:
- **Preserves correctness**: Still uses the highest severity (no logic change)
- **Maximizes visibility**: Conflict is in its own dedicated section, impossible to miss
- **Provides context**: Example shows how Big Head explains the discrepancy
- **Enables action**: Queen can follow up with reviewers if calibration is a pattern
- **Aligns with principles**: Extends "full traceability" to include severity assessment variations
