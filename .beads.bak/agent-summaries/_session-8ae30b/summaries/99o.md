## Selected Approach

**Choice**: Approach 5 (Hybrid – Expansion + Checklist)

**Rationale**: The root cause explicitly states that "a fresh Pantry agent reading only pantry.md would not know whether this read is informational or extractive." This dual uncertainty requires a solution addressing both aspects simultaneously.

- Approach 1 (minimal) fails the acceptance criterion—the agent would still be guessing about purpose.
- Approach 2 (paragraph) explains context but lacks verification targets.
- Approach 3 (annotation) is tightly coupled to implementation.md and risks staleness.
- Approach 4 (checklist) provides targets but lacks motivating context.
- Approach 5 combines narrative clarity with concrete extraction targets. The context cost is justified because Step 1 is foundational—misunderstanding here cascades to inadequate task briefs downstream.

The acceptance criteria explicitly require: "A fresh Pantry agent reading only pantry.md would know exactly how implementation.md shapes its output." Only Approach 5 satisfies this demand for both the "why" (narrative) and the "what" (concrete checklist).
