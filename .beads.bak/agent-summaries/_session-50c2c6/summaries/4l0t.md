**Task**: Add PARTIAL verdict state to checkpoints.md Verdict Thresholds Summary
**Status**: COMPLETE

## Root Cause

The Verdict Thresholds Summary section in `orchestration/templates/checkpoints.md` defined PASS, WARN, and FAIL but omitted PARTIAL. PARTIAL is used in the DMVDC and CCB checkpoint sections (both in the Detail by Checkpoint prose and in the Verdict blocks of the inline prompt templates). Pest Control agents reading the summary section first would not expect PARTIAL when they encounter it later in the file.

## Files Changed

- `orchestration/templates/checkpoints.md`

## Implementation

Two edits were made, both within the "Verdict Thresholds Summary" section:

**1. Common Verdict Definitions (around line 50)**

Added PARTIAL as a named verdict state between WARN and FAIL:

```
**PARTIAL** (DMVDC and CCB only): Some checks failed. Agent can repair and resubmit, or consolidation can be amended. Does not escalate to user.
```

Also updated the opening sentence from "All checkpoints use three verdict states:" to "All checkpoints use the following verdict states:" to avoid the now-incorrect count.

**2. Checkpoint-Specific Thresholds table (around line 70)**

Updated the Queue Blocking column for DMVDC and CCB rows to reference PARTIAL instead of WARN:

- DMVDC (Dirt Pushers): "PARTIAL allows resubmission; FAIL escalates"
- DMVDC (Nitpickers): "PARTIAL allows resubmission; FAIL escalates"
- CCB: "PARTIAL: fix and re-run; FAIL blocks user presentation"
