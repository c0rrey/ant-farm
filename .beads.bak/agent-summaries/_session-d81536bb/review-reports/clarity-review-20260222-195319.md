## Findings Catalog

### Finding 1: SESSION_PLAN_TEMPLATE.md uses emoji-based risk labels inconsistent with rest of codebase
- **File(s)**: `orchestration/templates/SESSION_PLAN_TEMPLATE.md:L45-L47`
- **Severity**: P3
- **Category**: clarity / consistency
- **Description**: The conflict risk assessment section uses emoji symbols (`🔴`, `🟡`, `🟢`) as risk level indicators. No other file in this codebase uses emoji for status signaling. RULES.md, GLOSSARY.md, and other templates use plain text or table notation. This is an intra-file inconsistency when the template is compared to surrounding documentation conventions.
- **Suggested fix**: Replace emoji risk labels with plain text equivalents matching the rest of the codebase: `HIGH RISK`, `MEDIUM RISK`, `LOW RISK`. If retained for readability in a planning document, add a note that these are visual aids only and will not be reproduced by orchestration automation.
- **Cross-reference**: N/A

### Finding 2: SESSION_PLAN_TEMPLATE.md "Estimated time" fields are incompatible with anti-pattern in RULES.md
- **File(s)**: `orchestration/templates/SESSION_PLAN_TEMPLATE.md:L64, L68, L72, L85, L90, L97, L105`
- **Severity**: P3
- **Category**: clarity / consistency
- **Description**: The template contains "Estimated time: X min" and "Total estimated time: X minutes" fields throughout Option A, B, and C strategy blocks. The broader system documentation explicitly discourages time estimates, and CLAUDE.md instructs agents to avoid giving time estimates. These fields in the template may confuse adopters who copy the template and fill in estimates that contradict the no-estimate guidance.

- **Cross-reference**: N/A

### Finding 3: `reviews.md` — Big Head polling loop has unsubstituted placeholder paths in shell code
- **File(s)**: `orchestration/templates/reviews.md:L531-L558`
- **Severity**: P3
- **Category**: clarity
- **Description**: The placeholder substitution guard loop (lines 531-558) checks for unresolved `<session-dir>` and `<timestamp>` angle-bracket placeholders in file paths. However, the actual path strings being checked in the loop body (`<session-dir>/review-reports/correctness-review-<timestamp>.md`, etc.) still contain raw angle-bracket placeholders — they are intentionally template text. A reader unfamiliar with the design intent could reasonably mistake these as examples that should have been substituted before shipping this file. The comment on L527 ("Placeholder substitution guard: verify the Pantry replaced all template placeholders") partially explains the intent but doesn't clarify that these literal strings in the guard are correct as written.
- **Suggested fix**: Add an inline comment immediately above or after each `_path` assignment in the guard loop that states: "These placeholder literals are intentional — the guard checks whether Pantry substituted them." For example: `# Intentional: compare actual path against this literal to detect Pantry substitution failure`.
- **Cross-reference**: N/A

### Finding 4: `reviews.md` — `REVIEW_ROUND={{REVIEW_ROUND}}` uses double-brace syntax inconsistently with single-brace used elsewhere
- **File(s)**: `orchestration/templates/reviews.md:L501`
- **Severity**: P3
- **Category**: clarity / consistency
- **Description**: The Big Head polling loop at line 501 uses `{{REVIEW_ROUND}}` (double braces) for the placeholder in the shell variable assignment. All other placeholders in the orchestration templates use single braces `{PLACEHOLDER}` (e.g., `{SESSION_DIR}`, `{TIMESTAMP}`, `{COMMIT_RANGE}`). The comment on L597-L598 explains that `fill-review-slots.sh` substitutes `{{REVIEW_ROUND}}`, suggesting double braces are intentional to distinguish script-substituted values from template text. However, there is no documented convention explaining the two-tier placeholder syntax. A developer reading this for the first time has no way to know why double braces appear here and nowhere else.
- **Suggested fix**: Add a comment near L501 (or in the section header) explaining: "Double braces `{{REVIEW_ROUND}}` indicate substitution by `fill-review-slots.sh`. Single braces `{PLACEHOLDER}` in template prose indicate Queen-filled values. These are two distinct substitution tiers."
- **Cross-reference**: N/A

### Finding 6: `RULES.md` — `{SESSION_DIR}/session-summary.md` entry does not clarify its purpose relative to `exec-summary.md`
- **File(s)**: `orchestration/RULES.md:L456`
- **Severity**: P3
- **Category**: clarity
- **Description**: The Session Directory section lists `session-summary.md` as "written by Pantry (optional)" alongside `exec-summary.md` (written by Scribe). Both are root-level session artifacts but RULES.md's Step 2 description of the Pantry does not mention session-summary.md, leaving a reader uncertain what it contains or when the "(optional)" condition applies. The Drift reviewer confirmed (post-review message) that session-summary.md is actively produced by the Pantry per `orchestration/templates/pantry.md:L154, L209` and coexists with exec-summary.md — they serve different purposes (Pantry planned-execution state vs. Scribe post-session canonical record). The issue is purely a clarity gap: RULES.md's Step 2 Pantry description does not cross-reference session-summary.md, so a reader of RULES.md alone cannot confirm the file is still live.
- **Suggested fix**: In the Session Directory artifact table, add a brief description distinguishing the two files: "session-summary.md — Pantry planned-execution state (see pantry.md); exec-summary.md — Scribe post-session canonical record (see scribe-skeleton.md)." Alternatively, add a note in the Step 2 Pantry description that it writes session-summary.md.
- **Cross-reference**: Drift reviewer confirmed no cross-file drift. This is a within-RULES.md description gap only.

### Finding 10: `parse-progress-log.sh` — UNREACHABLE comment at L216-L218 is accurate but slightly misleading
- **File(s)**: `scripts/parse-progress-log.sh:L216-L218`
- **Severity**: P3
- **Category**: clarity
- **Description**: The comment reads "UNREACHABLE: RESUME_STEP is always set by the loop above because SESSION_COMPLETE is in STEP_KEYS. ## Preliminary Groupings

### Group A: Template placeholder and substitution convention underdocumented
- Finding 3, Finding 4 — both concern the reviews.md Big Head polling section where placeholder conventions are opaque without inline explanation. The root cause: the two-tier substitution system (single braces for Queen-filled, double braces for script-filled, intentional angle-bracket literals for guards) is a design decision that lives only in the code, not in any documentation.
- **Suggested combined fix**: Add a brief convention note at the top of the polling loop section (or point to a dedicated `PLACEHOLDER_CONVENTIONS.md` already referenced in `README.md:L358`) explaining the three tiers.

### Group B: Optional/deprecated artifact entries without enough inline context
- Finding 6 (`session-summary.md` entry does not distinguish its role from exec-summary.md), Finding 9 (`pantry-review` deprecated pointer omits the `scripts/` path prefix) — both entries leave the reader without enough information to understand current artifact status without consulting a second file.
- **Note on Finding 6**: Drift reviewer confirmed session-summary.md is actively produced (not deprecated). The group root cause is still valid: RULES.md Step 2 omits a cross-reference to pantry.md where session-summary.md is described.
- **Suggested combined fix**: For each optional/deprecated artifact entry in RULES.md and README.md, add the canonical source file or path that defines current behavior, so readers don't need to chase external references.

### Group C: SESSION_PLAN_TEMPLATE.md convention mismatches with system norms
- Finding 1 (emoji risk labels), Finding 2 (time estimate fields) — both are within SESSION_PLAN_TEMPLATE.md and both conflict with system conventions (no emoji, no time estimates). Since this file is intended to be copied and filled in, convention mismatches here are likely to propagate into user-created plans.
- **Suggested combined fix**: Review the template for any remaining convention mismatches against RULES.md and CLAUDE.md guidelines, then strip or annotate the conflicting sections.

### Group D: Diagram and table incompleteness in README.md
- Finding 8 (Pest Control box missing SSV/ESV), Finding 9 (pantry-review deprecated note incomplete) — both are README.md documentation gaps that create a partial picture of the current system state.
- **Suggested combined fix**: Do a single-pass review of the README architecture diagram and agents table against current RULES.md checkpoint list and agents directory to catch any other omissions.

### Group E: Standalone findings (no group)
- Finding 5 (checkpoints.md internal task ID references — generalize or cross-reference)
- Finding 7 (scribe-skeleton.md step header names could include artifact names)
- Finding 10 (parse-progress-log.sh unreachable branch fallback value)
- Finding 11 (GLOSSARY.md priority annotations need context)
- Finding 12 (queen-state.md Source of Truth column mixes abstraction levels)
