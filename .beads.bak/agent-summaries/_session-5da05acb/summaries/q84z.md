# Summary: ant-farm-q84z
## Dual TIMESTAMP/REVIEW_TIMESTAMP naming convention creates cognitive burden

---

## 1. Approaches Considered

**Approach A: Unify on `TIMESTAMP` — drop `{REVIEW_TIMESTAMP}` from prose**
Remove the three-line mapping explanation (L147-149) and replace the prose identifier `{REVIEW_TIMESTAMP}` with `{TIMESTAMP}`. The bash snippet already uses `${TIMESTAMP}`, so only prose changes. This is the minimal, least-invasive fix. No other files touched.

**Approach B: Unify on `REVIEW_TIMESTAMP` — rename shell var**
Keep `{REVIEW_TIMESTAMP}` in prose and rename the shell variable from `TIMESTAMP` to `REVIEW_TIMESTAMP` everywhere in the bash snippets. More descriptive name but makes bash more verbose and risks confusion with the existing `REVIEW_ROUND` variable.

**Approach C: Delete the mapping explanation without changing any names**
Remove only the three-line explanation paragraph while leaving both `${TIMESTAMP}` and `{REVIEW_TIMESTAMP}` in place. Does not satisfy acceptance criterion 1 (only one name). Rejected.

**Approach D: Introduce a third canonical name (`{SESSION_TIMESTAMP}`) and update both**
Rename everything to a new, unambiguous term. Maximally clear but introduces a new name that is not currently used anywhere, increasing the scope of the change and risking inconsistency with external references. Rejected as over-engineering.

---

## 2. Selected Approach with Rationale

**Approach A selected.**

The bash snippet already uses `${TIMESTAMP}` and is correct. The only problem was the prose introducing a second name `{REVIEW_TIMESTAMP}` and then requiring readers to mentally map between the two. Removing the three-line dual-naming explanation and replacing it with a single concise line that states both the placeholder form `{TIMESTAMP}` and the shell form `${TIMESTAMP}` eliminates the cognitive burden with the fewest characters changed.
