# Summary: ant-farm-eq77

## Approaches Considered

1. **Add a deployment note in SETUP.md** — In the Quick Setup section, add a note explaining that `code-reviewer` is a custom Claude Code agent that lives in `~/.claude/agents/` (not in the repo's `agents/` directory) and must be copied manually when setting up on a new machine. This is the minimal documentation-only change that satisfies both acceptance criteria.

2. **Add a note in SETUP.md AND add a row to the Agent Types table in RULES.md** — Document the agent in both locations. Tradeoff: the Agent Types table in RULES.md documents Queen-spawned agents; `code-reviewer` is not spawned by the Queen but by Nitpicker team members. Adding it there could be misleading about the spawning hierarchy.

3. **Add a note in SETUP.md AND add a comment in checkpoints.md or reviews.md** — reviews.md already references `code-reviewer` as "Agent Type" for each review section; adding a note there alongside those references would be highly discoverable. Tradeoff: reviews.md is read by the Pantry (not new adopters), so SETUP.md is a better location for onboarding-facing documentation.

4. **Move code-reviewer.md into the repo's agents/ directory** — This would make sync-to-claude.sh handle deployment automatically. Tradeoff: the task brief explicitly states "documentation-only task" and "Do NOT edit: ~/.claude/agents/code-reviewer.md, scripts/sync-to-claude.sh, or any code files." Moving the agent file is outside the allowed scope.

## Selected Approach

**Add a deployment note in SETUP.md** (Approach 1).

Rationale: SETUP.md is the canonical onboarding document. New adopters following the Quick Setup will encounter this note immediately after the sync step, before they encounter the review pipeline that requires code-reviewer. The task is explicitly documentation-only. DRIFT-002 (ant-farm-h94m) established that Pest Control runs checkpoints directly without spawning code-reviewer subagents; DRIFT-020 (this task) specifically targets the undocumented deployment gap. A SETUP.md note satisfies both acceptance criteria without touching the agent file or scripts.

## Implementation Description

Added a note block to SETUP.md in the Quick Setup section, immediately after the bash block for Step 1 (install hook and sync) and before Step 2 (add orchestration reference to project CLAUDE.md).

The note explains:
1. What `code-reviewer` is (custom Claude Code agent type used by Nitpicker reviewers)
2. Why sync-to-claude.sh does not deploy it (lives in global `~/.claude/agents/`, not repo `agents/`)
3. What new adopters must do (copy/create `~/.claude/agents/code-reviewer.md` manually)
4. Where to find the source file on the original machine
5. The consequence of not having the file (Nitpicker team members fail to spawn)

The Agent Types table in RULES.md was not modified because it documents Queen-spawned agents; `code-reviewer` is a Nitpicker team member type, not a Queen-spawned agent.
