# Summary: ant-farm-q59z — Big Head CCB Timeout Fix

**Commit**: a59d61e
**Status**: CLOSED

## 1. Approaches Considered

**Approach A — Pure turn-based, no retry mechanism**
Remove all timeout/retry language entirely. Big Head sends to Pest Control and ends its turn. No follow-up if Pest Control fails to respond. Simplest diff possible, but eliminates the escalation safety net entirely. If Pest Control genuinely crashes or stalls, the workflow hangs silently with no recovery path.

**Approach B — Turn-count retry with escalation (selected)**
Replace wall-clock seconds with turn counts. Big Head sends to Pest Control and ends its turn immediately. If no reply after 2 subsequent turns of other work, send one retry. After 2 more turns with no reply, escalate to Queen. Preserves the escalation safety net with no sleep calls. Exactly matches the fix specification in the bug description.

**Approach C — Marker-file polling on subsequent turns**
Big Head writes a "waiting-for-PC" marker file after sending the SendMessage and ends its turn. On subsequent turns, checks whether the marker has been replaced by a Pest Control result file. Avoids changing the retry language but introduces file-system coupling between two agents and requires Pest Control to write to a Big Head-owned path — an undocumented side channel.

**Approach D — Queen relay**
Big Head sends to Pest Control via Queen. Queen relays the message and relays the response back. Eliminates the timeout issue by making Queen responsible for coordination. High coupling, adds Queen context cost on every review round, and is precisely the "Queen intervention" the fix is designed to eliminate.

---

## 2. Selected Approach and Rationale

**Approach B — turn-count retry.**

Rationale:
- Directly eliminates the root cause: no sleep calls means Big Head's turn ends and incoming messages can be processed.
- Minimal surface area — only the two specified sections change.
- Preserves the escalation safety net so a genuinely unavailable Pest Control is still surfaced to the Queen.
- Uses turn counts (2 turns, then 2 more) as the natural unit of "time" in a turn-based message system.
- Matches the exact resolution described in the bug's "Fix" section, reducing risk of misalignment.
