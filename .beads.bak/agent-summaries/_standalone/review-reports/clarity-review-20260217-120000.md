### Finding 6: Checkpoint A Nitpickers section says "7 checks" but lists checks numbered 0-6

- **File(s)**: `/Users/correy/.claude/orchestration/templates/checkpoints.md:154`

- **Description**: Line 154 says "All 7 checks pass for all 4 prompts" but the checklist items are numbered 0 through 6 (which is indeed 7 items). ### Finding 8: checkpoints.md uses "ceil(N/3)" without explaining rounding behavior

- **File(s)**: `/Users/correy/.claude/orchestration/templates/checkpoints.md:311`
- **Severity**: P3
- **Category**: clarity / readability
- **Description**: Line 311 says "Pick `min(5, ceil(N/3))` random findings". While this is a clear mathematical formula, the surrounding context uses plain English for all other instructions. The formula is compact but may cause a model to miscalculate (e.g., interpreting "ceil" as something else). The same line also says "minimum 3, or all findings if fewer than 3" which partially restates the formula in English but doesn't fully match -- the formula gives min(5, ceil(N/3)) which for N=6 gives min(5,2)=2, but the text says "minimum 3".
- **Suggested fix**: Clarify the sampling rule in plain English. For example: "Pick at least 3 findings (or all findings if fewer than 3 exist), up to a maximum of 5." This resolves the contradiction between the formula (which can produce 2) and the text (which says minimum 3).
- **Cross-reference**: This is arguably an edge-case/correctness issue -- the formula and the English description contradict each other. Flagging for the edge-cases reviewer.
