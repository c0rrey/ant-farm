# ant-farm-12u9 — Fix Summary

## Issue
Missing `<IF ROUND 1>` markers on the placeholder guard's `for _path in` block in
`orchestration/templates/reviews.md`. The guard iterated over all 4 review paths
(correctness, edge-cases, clarity, excellence) in a single for-loop, but the
clarity/excellence paths had no `# <IF ROUND 1>` / `# </IF ROUND 1>` markers.
In round 2+, Pantry strips marked lines but leaves unmarked ones, so the unsubstituted
clarity/excellence placeholder paths remained in the guard, triggering a false-positive
`PLACEHOLDER_ERROR=1` and `exit 1`.

## Root Cause
The original for-loop used a single continuation-line list (`for _path in \ ... ; do`).
Shell comment lines can't appear inside such a list, so the `<IF ROUND 1>` markers
couldn't be applied inline. The clarity/excellence paths were simply left unguarded.

## Fix
Split the single for-loop into two loops:
1. A loop for the always-expected paths (correctness, edge-cases) — no markers needed.
2. A loop for the round-1-only paths (clarity, excellence) — wrapped with:
   ```
   # <IF ROUND 1>
   for _path in \
     "<session-dir>/review-reports/clarity-review-<timestamp>.md" \
     "<session-dir>/review-reports/excellence-review-<timestamp>.md"; do
     ...
   done
   # </IF ROUND 1>
   ```

This matches the `# <IF ROUND 1>` / `# </IF ROUND 1>` pattern already used in the
while-loop body at lines 566-569.
