# ant-farm-l3d5 — Template placeholder substitution not validated at consumption point

**Status**: CLOSED
**Commit**: fix: add placeholder substitution validation guards at consumption points (ant-farm-l3d5)

## Changes Made

### orchestration/templates/reviews.md (lines ~515-537)
Added a placeholder-presence guard in the bash polling loop block, inserted before the `while` loop. The guard iterates over all four expected report file paths and uses a `case` statement to detect any path that still contains angle brackets (`<`, `>`) or curly braces (`{`, `}`). If any unsubstituted placeholder is found, it prints a diagnostic message identifying the path and naming the root cause (upstream Pantry substitution failure), sets `PLACEHOLDER_ERROR=1`, and exits 1 before the polling loop ever runs.

This prevents the previous failure mode where unsubstituted paths caused every `[ -f ]` test to silently return false, producing a 30-second timeout and a generic "TIMEOUT: Not all expected reports arrived" error with no indication of the real problem.

### orchestration/templates/checkpoints.md (lines ~199-201)
Improved the CCO input guard abort message. The original message was a single terse line that echoed the placeholder name without explaining why the value was invalid. The new message:
- Explicitly names the scenario where `{REVIEW_ROUND}` is still a literal placeholder (curly braces present)
- States the root cause: upstream substitution failure in the Queen/Pantry prompt composition step
- Provides a concrete fix: fill in REVIEW_ROUND as a plain integer before spawning Pest Control

## Impact
Both fixes are consumption-point guards — they catch substitution failures at the earliest possible moment and produce actionable diagnostic output instead of cryptic timeouts or opaque abort messages.
