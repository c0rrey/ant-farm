# Task Summary: ant-farm-1b8

## Task
Installation guide uninstall uses wrong path `~/.git/hooks/pre-push` instead of `.git/hooks/pre-push`. Also, uninstall instructions only cover the pre-push hook; the pre-commit hook has no uninstall instructions.

## Approaches Considered

### Approach A — Fix path only (minimal)
Change `rm ~/.git/hooks/pre-push` to `rm .git/hooks/pre-push` on L193 alone. Satisfies acceptance criterion 1 but does not address criterion 2 (missing pre-commit uninstall instructions).

### Approach B — Fix path and add pre-commit as a comment
Fix the path and add `# rm .git/hooks/pre-commit` as a commented example in the same block. Minimal diff, but shell comments on `rm` commands are confusing — readers may not know whether to uncomment and run.

### Approach C — Fix path and add pre-commit as an active command (SELECTED)
Fix `rm ~/.git/hooks/pre-push` to `rm .git/hooks/pre-push`, add an explicit active `rm .git/hooks/pre-commit` line with a descriptive comment, and update the restore-from-backup lines to include both hooks. Add a clarifying comment on the pre-push line explaining the repo-relative path. Update the Verification block to check both hooks. Satisfies both acceptance criteria with minimal scope expansion.

### Approach D — Fix path and split into per-hook subsections
Fix the path and restructure Step 1 into two subsections ("Pre-Push Hook" and "Pre-Commit Hook") with separate remove and restore blocks for each. Most thorough, but larger diff and introduces subsection structure that may conflict with the document's established style for the Uninstall section.

## Selected Approach

**Approach C** was chosen because it satisfies both acceptance criteria within the narrowest scope (L185-295). The single code block already had a "restore from backup" example; adding parallel `rm` and `cp` lines for the pre-commit hook is natural and consistent. A brief comment on the pre-push line ("runs from repo root — not from home directory") directly calls out the bug that caused the issue, reducing the chance of future regression. Approach D was rejected because the additional subsection structure wasn't needed to satisfy the criteria.
