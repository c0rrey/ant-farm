# Summary: ant-farm-z69
**Task**: Pre-push hook blocks all git pushes when sync-to-claude.sh fails

## 1. Approaches Considered

**A. Explicit `if ! "$SYNC_SCRIPT"; then warn; fi` (selected)**
- Wraps the sync call in a conditional; `if` swallows the exit code from `set -e` perspective
- Warning message is emitted to stderr; hook continues to implicit `exit 0`
- Missing/non-executable guard (exit 1) is unaffected
- Tradeoff: slightly more verbose than an OR-chain but maximally readable

**B. OR-chain: `"$SYNC_SCRIPT" || echo "[ant-farm] WARNING: sync failed"`**
- Very concise; the `||` idiom is idiomatic bash
- Compatible with `set -euo pipefail` because `||` is a compound list, not a simple command
- Tradeoff: no structured block to add multi-line warning in the future without refactoring

**C. Remove `set -euo pipefail` from the hook entirely**
- Simplest diff — one line deleted
- Tradeoff: removes protection for the `[[ ! -x ]]` guard — if the guard's `exit 1` were accidentally removed, there would be no safety net. Also removes pipefail protection from other potential future additions.

**D. Subshell isolation: `( "$SYNC_SCRIPT" ) || echo WARNING`**
- Runs sync in a subshell so its exit code cannot escape to the parent hook process
- Tradeoff: subprocess overhead, harder to reason about variable scope, overkill for this use case

## 2. Selected Approach with Rationale

Approach A: `if ! "$SYNC_SCRIPT"; then ... fi`

- **Correctness**: The `if` statement evaluates the exit status of its condition command and does NOT trigger `set -e` on non-zero — this is specified by POSIX and bash behavior. The hook exits 0 after the if block, allowing the push.
- **Preserves guard**: The `[[ ! -x "$SYNC_SCRIPT" ]]` check at L39-41 still uses `exit 1`, correctly blocking pushes when the script is missing or non-executable.
- **Readable**: The warning message clearly explains what happened and that the push continues.
- **Minimal diff**: Two lines changed in the hook template body.

## 3. Implementation Description

One change to `scripts/install-hooks.sh` in the pre-push hook template body (L44-46):

Before:
```bash
"$SYNC_SCRIPT"
```

After:
```bash
if ! "$SYNC_SCRIPT"; then
    echo "[ant-farm] WARNING: sync-to-claude.sh failed — push continuing without sync." >&2
fi
```

The `if !` pattern is the standard bash idiom for running a command and branching on failure without triggering `set -e`. No other changes made.
