# ant-farm-ul02 — Bug Fix Summary

## Issue

P1 BLOCKER: `extract_agent_section` in `scripts/compose-review-skeletons.sh` used
`count>=2` in its awk pattern, requiring TWO `---` delimiters before emitting any
output. Skeleton template files (`nitpicker-skeleton.md`, `big-head-skeleton.md`)
only have ONE `---` separator between the human-readable instruction block and the
agent-facing template body. Result: the function silently produced empty output,
causing all five skeleton files to contain only the composed header comment and
the Review Brief / Consolidation Brief footers — with no agent instructions in
between.

## Root Cause

Prior fix ant-farm-o058 introduced `count++; count>=2` under the assumption that
skeleton files follow YAML frontmatter conventions (opening `---`, content,
closing `---`). The actual structure is:

```
[Human-readable instruction block]
---
[Agent-facing template body]
```

Only one `---` delimiter exists. The `count>=2` guard consumed that delimiter
and then found no second one, so `count` never reached 2 and nothing was printed.

## Fix Applied

`scripts/compose-review-skeletons.sh` line 73:

Before:
```awk
awk '/^---$/{count++; next} count>=2{print}' "$file"
```

After:
```awk
awk '/^---$/{count++; next} count>=1{print}' "$file"
```

The function's documented purpose is "everything after the line containing only
`---`". `count>=1` correctly implements that — once the first `---` is consumed
(`count` becomes 1), all subsequent lines are printed.
