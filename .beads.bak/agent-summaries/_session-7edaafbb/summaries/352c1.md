**Approach A: Shell `if` conditional with `{{REVIEW_ROUND}}` slot (selected)**
Replace `# <IF ROUND 1>` / `# </IF ROUND 1>` with `if [ "$REVIEW_ROUND" -eq 1 ]; then ... fi`. **Approach B: Script-side awk/sed strip pass before brief delivery**

- Pro: Template stays conceptually "flat" after delivery.

**Approach C: Two separate template variants (round-1 and round-N)**
Maintain `big-head-round1.md` and `big-head-roundN.md`; fill-review-slots.sh picks the appropriate one.
- Pro: No conditionals anywhere.

**Approach D: Emit round-specific shell code via a `{{ROUND1_GUARD_BLOCK}}` slot**
Define `{{ROUND1_GUARD_BLOCK}}` as an empty string in round 2+ and as the full guard for loop in round 1. Fill via script.
- Pro: Template has a single substitution point.
- Con: Multi-line slot values are more complex to substitute; the block is opaque to readers of the template.

## 2. Selected Approach with Rationale

Approach A. The template bash block is already designed to be executed by Big Head in a single Bash invocation. Using a real shell `if` conditional makes the round-gating executable by the shell interpreter — not the LLM. The `{{REVIEW_ROUND}}` slot is already filled by `write_big_head_brief()` in fill-review-slots.sh (L313), so no script changes are needed. This is the minimal, highest-clarity fix.
