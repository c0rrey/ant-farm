## 1. Approaches Considered

**Approach A: Post-write grep scan in existing verification section (selected)**
In the verification loop at L350-385 of fill-review-slots.sh, add `elif grep -qE '\{\{[A-Z_]+\}\}' "$f"` after the non-empty check. Also extend to cover the Big Head brief. Unfilled markers are reported with their specific names using `grep -oE`.
- Pro: Minimal change, co-located with existing file checks, applies to all output files uniformly.
- Con: Only catches `{{UPPERCASE}}` patterns; does not catch angle-bracket placeholders like `<session-dir>`. However, `<...>` placeholders in file paths are handled by Big Head's own bash placeholder guard in reviews.md.

**Approach B: Per-slot validation inside fill_all_slots**
After awk substitution, re-scan the file for each slot name from the map to confirm replacement occurred.
- Pro: Validates precisely the slots that were supposed to be substituted.
- Con: fill_all_slots is already complex (temp files, mapfile, awk). Adding a re-scan per slot makes it harder to reason about. Overkill for the failure mode described.

**Approach C: Pre-fill validation — scan skeleton for unknown slots**
Before substitution, parse the skeleton for `{{...}}` markers and warn if any are not in the provided slot list.
- Pro: Early failure before writing anything.
- Con: Requires parsing skeleton format; some markers may be intentional (e.g., documentation examples inside code blocks). False positives are a real risk.

**Approach D: Write-time atomic check inside awk**
Extend the awk script in fill_all_slots to emit an error if any `{{` sequence remains after applying all substitutions.
- Pro: Integrated with substitution.
- Con: Requires threading error state out of awk back to bash; more fragile than a simple grep pass.

## 2. Selected Approach with Rationale

Approach A. The verification section (L345-373) already checks existence and non-empty. Adding a `grep -qE '\{\{[A-Z_]+\}\}'` check as an `elif` branch fits naturally into the existing if/elif/fi pattern. The `grep -oE` pass with `sort -u` provides clear diagnostic output (which specific markers were not filled). Using `grep -E` (POSIX extended regex) avoids the macOS incompatibility with `grep -P` (Perl regex not supported on macOS BSD grep).

## 3. Implementation Description

Changes to `scripts/fill-review-slots.sh` (L350-381):

1. Review prompt/preview files loop (L351-370): Added `elif grep -qE '\{\{[A-Z_]+\}\}' "$f"; then` branch after the `elif [ ! -s "$f" ]` check. On match, prints an error with the file path and lists each unique unfilled marker via `grep -oE '\{\{[A-Z_]+\}\}' "$f" | sort -u`.

2. Big Head brief check (L372-381): Extended from a simple `if [ ! -f ... ]` to an `if/elif` chain. The new `elif` branch applies the same unfilled-slot grep to the Big Head consolidation brief.

Pattern used: `\{\{[A-Z_]+\}\}` — matches `{{` followed by one or more uppercase letters or underscores, followed by `}}`. This matches the `{{SLOT_NAME}}` convention used throughout the codebase while avoiding false positives on lowercase or mixed-case content in template prose.

No changes to `orchestration/templates/reviews.md` for this task. The conditional block robustness (AC1) was addressed by commit 1906d23 (ant-farm-352c.1), which replaced `# <IF ROUND 1>` markers with `if [ "$REVIEW_ROUND" -eq 1 ]; then ... fi` shell conditionals.
