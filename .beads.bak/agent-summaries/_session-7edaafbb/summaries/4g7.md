**Task**: install-hooks.sh generated hook lacks descriptive error on sync script failure
**Commit**: 54ffeed

## 1. Approaches Considered

**Approach A — Additional echo lines with Cause and Fix labels (selected)**
Add two more `echo` lines within the failure if-block. One explains the cause ("the sync script exited non-zero"), the other provides a remediation step ("run scripts/sync-to-claude.sh manually"). Minimal diff, no structural change.

**Approach B — Capture and print sync script exit code**
Store `sync_exit=$?` before the if-block and interpolate it into the warning. Adds numeric diagnostic info but exit codes without context don't help most operators know what to do next.

**Approach C — Redirect sync stderr to temp file, print on failure**
Capture the script's own error output and replay it in the warning block. Most informative, but adds temp file management complexity and changes the hook's execution model significantly.

**Approach D — Multi-line printf with a formatted box**
Use `printf` with `\n` for a visually distinct block. More eye-catching but adds complexity inside an already heredoc-in-heredoc context and the extra indirection has no functional benefit.

## 2. Selected Approach

Approach A. Two additional `echo` lines inside the existing failure if-block deliver both a cause description and a clear remediation step with zero structural change to the heredoc or execution model. The `[ant-farm]` prefix and `>&2` redirection are preserved for consistency with the surrounding code.

## 3. Implementation Description

Added two lines inside the `if ! "$SYNC_SCRIPT"; then` block within the `cat > "$HOOK_TARGET" <<'HOOK'` heredoc (L44-48):

```bash
if ! "$SYNC_SCRIPT"; then
    echo "[ant-farm] WARNING: sync-to-claude.sh failed — push continuing without sync." >&2
    echo "[ant-farm]   Cause: the sync script exited non-zero (check its output above for details)." >&2
    echo "[ant-farm]   Fix:   run 'scripts/sync-to-claude.sh' manually to diagnose, then re-push." >&2
fi
```

These lines appear verbatim in the generated `.git/hooks/pre-push` file because the heredoc delimiter is quoted (`<<'HOOK'`).
