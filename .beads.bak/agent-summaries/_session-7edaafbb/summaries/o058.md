## Approaches Considered

1. **Count-based awk (selected)**: Track `---` occurrences with a counter; print only when `count>=2`. `awk '/^---$/{count++; next} count>=2{print}'`. Minimal one-line change, expresses intent clearly, handles the YAML frontmatter pair contract directly.

2. **State-machine awk with explicit flags**: Use two boolean flags `in_frontmatter` and `body_started`. More lines, more explicit state transitions. Correct but verbose for what is a simple counting problem. No benefit over the counter approach.

3. **Python/perl one-liner**: Replace awk with `python3 -c` or `perl -ne` for more readable logic. Introduces interpreter dependency not guaranteed on all target systems (though Python3 is common). Overkill for a two-state skip problem.

4. **Sed two-pass approach**: Use `sed '1,/^---$/d'` twice in a pipe to remove everything up through the second `---`. Achieves the result but is less readable, requires two sed invocations, and the idiom is less immediately obvious than counting with awk.

## Selected Approach with Rationale

Approach 1 (count-based awk) was selected because:
- It is the minimal, surgical change to the existing awk one-liner
- The counter idiom is immediately understandable: print only after two `---` delimiters have been seen
- No new dependencies or tool changes
- Directly matches the fix suggested in the bug report (`bd show` output)

## Implementation Description

Changed line 73 of `scripts/compose-review-skeletons.sh`:

**Before:**
```bash
awk '/^---$/{found=1; next} found{print}' "$file"
```

**After:**
```bash
awk '/^---$/{count++; next} count>=2{print}' "$file"
```

The `count` variable starts at 0 (awk default). Each `---` line increments it and is skipped (`next`). Non-separator lines are printed only when `count>=2`, meaning the opening frontmatter `---` has been seen (count=1) AND the closing frontmatter `---` has been seen (count=2). Content between the two delimiters (YAML fields) is correctly skipped.
