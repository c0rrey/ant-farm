## Approaches Considered

1. **Single awk pass with all values via `-v` flags** — Pass each slot value as an awk `-v` variable. Problem: awk `-v` cannot handle multiline values; newlines in the value string break the variable assignment. Rejected due to correctness failure on multiline CHANGED_FILES.

2. **Python subprocess single-pass replacement** — Use Python's `str.replace()` in a loop for all substitutions in one call. Reliable for multiline, but adds a mandatory Python dependency (the original code had Python as an optional fallback). Rejected to avoid changing the dependency profile.

3. **Chained sed `-e` expressions** — Pre-escape all values and chain multiple `-e s/slot/value/g` expressions in one sed invocation. Fragile: ampersands, backslashes, and newlines in replacement strings require complex escaping. Rejected as error-prone with the data patterns present.

4. **Map file approach: write slot→tmpfile index, single awk pass reads map** — Write each value to a separate temp file, then write a map file containing `slot_name TAB tmpfile_path` entries. One awk invocation per output file reads the map in BEGIN, loads all values, then applies all substitutions in a single pass over the file. Handles multiline values correctly, preserves the existing awk logic, and is auditable.

5. **Associative array accumulation in shell, build one awk script string** — Accumulate all slot/value pairs in shell arrays and dynamically build an awk program string. Risk of shell quoting escapes corrupting values with quotes or backslashes. Rejected as less readable than the map-file approach.

## Selected Approach

**Approach 4 — Map file + single awk pass** (implemented as `fill_all_slots`).

Rationale: Correctly handles multiline values (same mechanism as original — values read line-by-line from temp files). One awk process per output file instead of one per slot. Keeps the proven index()/substr() substitution logic unchanged. Clean cleanup of all temp files in one `rm -f` call.

## Implementation Description

Replaced `fill_slot` (single slot, single awk invocation) with `fill_all_slots` (all slots, single awk invocation):

- `fill_all_slots <file> <slot1> <val1> [<slot2> <val2> ...]` accepts variadic slot/value pairs
- Writes each value to a mktemp file; appends `slot\ttmpfile\n` to a shared map file
- Runs one awk process: BEGIN reads the map file and loads all values; body loop applies every substitution per line
- Cleans up map file and all value temp files with a single `rm -f` call
- Updated `write_filled_review` (7 slots) and `write_big_head_brief` (5 slots) to use `fill_all_slots`
