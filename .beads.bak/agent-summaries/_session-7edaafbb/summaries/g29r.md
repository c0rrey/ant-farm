# Task Summary: ant-farm-g29r

**Task**: sync-to-claude.sh silently skips missing source scripts
**Commit**: 9bba01f

## 1. Approaches Considered

1. **Warn to stderr then continue** — replace `|| continue` with an `if/warn/continue` block that emits a message to stderr before skipping. Minimal change, direct, correct channel for diagnostic messages.

2. **Fail the script on any missing script** — replace `|| continue` with `|| { echo ... >&2; exit 1; }`. Enforces presence of all expected scripts but is too strict for minimal installs or forks where a script may legitimately be absent.

3. **Pre-validate all scripts before entering the loop** — accumulate missing scripts into an array, print a single summary warning, then loop only over existing ones. More complex, same user-visible outcome as option 1.

4. **Dynamic discovery via glob instead of explicit list** — use a glob pattern to find any matching scripts, removing the need to check for specific names. Loses the ability to detect when an expected-but-missing script is absent, which is the entire point of the check.

## 2. Selected Approach

**Option 1: Warn to stderr, then continue.**

Rationale: Minimal, targeted change that exactly satisfies both acceptance criteria. The `>&2` redirect ensures the warning goes to the correct diagnostic channel, not stdout. The loop continues so present scripts are not affected.

## 3. Implementation Description

Changed `/Users/correy/projects/ant-farm/scripts/sync-to-claude.sh` lines 32-35:

Before:
```sh
    [ -f "$script" ] || continue
```

After:
```sh
    if [ ! -f "$script" ]; then
        echo "[ant-farm] WARNING: expected script not found, skipping: $script" >&2
        continue
    fi
```

The warning includes the full path of the missing script to make it actionable for the operator.
