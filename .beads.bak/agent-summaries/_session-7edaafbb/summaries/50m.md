# Task Summary: ant-farm-50m
**Task**: scrub-pii.sh assumes perl is installed without checking
**Commit**: 79b5ad3

## 1. Approaches Considered

**A — `command -v perl` guard after `set -euo pipefail` (selected)**
Shell-native, POSIX-compatible. `command -v` is the conventional idiom for dependency checks in bash scripts. Fails immediately before any file I/O with a descriptive, actionable error message.

**B — `which perl` check**
`which` is not POSIX; it may behave differently on minimal Alpine/BusyBox environments where the script is most likely to fail. `command -v` is the preferred bash idiom.

**C — `type -p perl` check**
Functionally equivalent to `command -v` but bash-specific and less conventional in portable shell scripts.

**D — Defer check to just before the perl invocation (L63)**
Only fails when perl is actually needed. By that point the check-mode path may have already exited 0 cleanly (check mode does not invoke perl), creating inconsistency. Startup checks are the preferred fail-fast pattern.

## 2. Selected Approach

Approach A — `command -v perl` guard at startup. It fires before any other logic, uses the conventional POSIX idiom, and prints a clear actionable error to stderr. The `--check` mode path does not invoke perl but the check is still useful to surface the dependency problem early (a missing perl would cause the next commit to fail silently at scrub time).

## 3. Implementation Description

Added four lines to `scripts/scrub-pii.sh` immediately after `set -euo pipefail` (now L17-20):

```bash
command -v perl >/dev/null 2>&1 || {
    echo "[scrub-pii] ERROR: perl is required but not found. Install perl and retry." >&2
    exit 1
}
```

Stdout from `command -v` is discarded; stderr is also suppressed. If perl is absent, a prefixed error message is printed to stderr and the script exits 1.
