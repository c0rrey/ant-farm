### FINDING D-3
**File**: `orchestration/templates/checkpoints.md:689,717`
**Severity**: P2
**Category**: Stale instruction — Pest Control's SSV verdict directs Queen to seek user approval

**Description**: The SSV checkpoint "Verdict" section in checkpoints.md instructs Pest Control to report PASS and then states:

> "The Queen will present the strategy to the user for approval before spawning Pantry — do NOT spawn Pantry yourself."

And the "The Queen's Response" subsection at line 717 states:

> "**On PASS**: Present the recommended strategy to the user for approval (Step 1b in RULES.md). **User approval is required even on SSV PASS — this is a deliberate design choice, not an omission.**"

This directly contradicts the updated RULES.md:L97-99:
> "**On SSV PASS**: Proceed directly to Step 2. Do NOT wait for user approval. SSV is the mechanical safety gate — a passing strategy is structurally sound and ready to execute."

**Impact**: Pest Control reads checkpoints.md to determine what to report after SSV PASS. The SSV PASS report it produces will echo the old guidance (seek user approval). When the Queen reads that report artifact, it receives contradictory signals from two authoritative sources. Additionally, any agent or developer consulting checkpoints.md for SSV behavior will implement the old user-approval flow. Confirmed as a real drift finding by the Correctness reviewer (see Cross-Review Messages).

**Severity rationale**: Downgraded from P1 to P2. The Queen is explicitly FORBIDDEN from reading checkpoints.md directly (RULES.md:L48), so the stale instruction does not reach the Queen via a direct read path. The risk is indirect: the SSV PASS report that Pest Control writes may echo the old wording, and the Queen reads that report. This is a real confusion risk but not a guaranteed runtime failure path.

**Suggested fix**: Update `orchestration/templates/checkpoints.md:689` and `:717` to reflect the auto-approve behavior:
- L689: "Report PASS to the Queen. The Queen will proceed directly to Step 2 — do NOT spawn Pantry yourself."
- L717: "**On PASS**: Proceed directly to Step 2 without waiting for user approval (per RULES.md Step 1b). SSV PASS is sufficient authorization."

**Note**: `checkpoints.md` is outside the scoped file list for this review. This finding is reportable because the contradiction would cause a Pest Control agent to produce a report instructing behavior that now conflicts with RULES.md — a cross-file assumption failure in a common execution path.
