# Summary: ant-farm-c05

**Task**: Checkpoint A.5 relies on Queen-provided file list with no independent scope validation

## 1. Approaches Considered

**Approach A: Document the limitation with a caveat note in Check 1.**
Add a "Known limitation" blockquote after Check 1's description explaining that the commit range is Queen-provided, with no independent fallback if the Queen passes incorrect hashes. Cross-reference WWD as a mitigation.
- Pros: Honest and transparent. Meets AC ("limitation is explicitly documented"). Minimal structural change.
- Cons: Does not add an independent validation mechanism.

**Approach B: Add independent validation -- Pest Control verifies the commit range.**
Add a pre-step where Pest Control runs `git log` to verify both commit hashes exist, the range is non-empty, and it spans the current session.
- Pros: Adds a real independent check against malformed commit ranges.
- Cons: Still cannot verify "correctness" of the range (whether it's too narrow/broad). Adds complexity. May exceed scope boundaries.

**Approach C: Cross-reference WWD as independent backstop only.**
Note that WWD independently validates scope per-task post-commit, so the system has coverage even if CCO's file list is wrong.
- Pros: Leverages existing infrastructure; shows system-level protection.
- Cons: WWD runs post-commit (not pre-spawn), so it cannot prevent bad reviews -- only detect scope creep afterward.

**Approach D: Add a "Scope Integrity" preamble to the template.**
Add a new section before the 7 checks documenting the full scope validation model: Queen provides commit range, Pest Control derives files via git diff, residual risk is accepted with WWD backstop.
- Pros: Comprehensive documentation.
- Cons: Adds structural weight to a template that should stay concise. Over-engineering for a single limitation note.

## 2. Selected Approach

**Approach A with elements of C** -- Add a "Known limitation" blockquote to Check 1 documenting the Queen-provided commit range dependency and cross-referencing WWD as post-commit mitigation.

Rationale: The acceptance criteria says "Either A.5 has an independent scope reference, or the limitation is explicitly documented." Since adding a truly independent scope reference is not feasible (Pest Control cannot independently determine the correct commit range), documenting the limitation is the appropriate choice. The WWD cross-reference shows the system-level mitigation.
