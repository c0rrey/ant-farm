# Fix Summary: ant-farm-35wk

## Issue

`scripts/parse-progress-log.sh` used `declare -A` (bash 4+ associative arrays) for three
maps: `STEP_COMPLETED`, `STEP_TIMESTAMP`, and `STEP_DETAILS`. macOS ships bash 3.2, which
does not support `declare -A`. Under `set -euo pipefail`, the `declare -A` call exits with
code 2. RULES.md Step 0 interprets exit code 2 as "session completed — start fresh", making
crash recovery completely broken on macOS. Additionally, when the script was invoked via zsh
(the macOS default and Claude Code shell), associative array lookups returned empty strings
even if the bash version happened to support them.

## Fix

Replaced the three `declare -A` associative arrays with a temp-directory-based key-value
store that is fully POSIX-compatible and works on bash 3.2, bash 5.x, and zsh.

### Approach

- `map_init` creates a `mktemp -d` directory with three subdirectories: `completed/`,
  `timestamp/`, and `details/`.
- `map_set <submap> <key> <value>` writes the value to a file at `<tmpdir>/<submap>/<key>`.
- `map_get <submap> <key>` reads and prints the file, or prints nothing if absent.
- `map_has <submap> <key>` tests file existence (`[ -f ... ]`), returning exit 0/1.
- A `trap 'map_cleanup' EXIT` ensures the temp directory is removed on normal exit, error,
  and signal.

All usage sites were updated:
- `STEP_COMPLETED["$key"]="yes"` → `map_set "completed" "$key" "yes"`
- `[ "${STEP_COMPLETED[$key]+set}" = "set" ]` → `map_has "completed" "$key"`
- `${STEP_TIMESTAMP[$key]}` → `$(map_get "timestamp" "$key")`
- `${STEP_DETAILS[$LAST_COMPLETED]:-}` → `$(map_get "details" "$LAST_COMPLETED")`
