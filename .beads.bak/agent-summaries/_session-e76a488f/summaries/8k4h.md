## 1. Approaches Considered

**Approach A: Sequential bead-by-bead deep dive**
Read each bead's description, locate the exact referenced code location, compare against current code for that single bead, assign verdict, move to next. Maximum accuracy per verdict but requires re-reading context for each script multiple times (e.g., reading parse-progress-log.sh 5 times for 5 beads that reference it).

Tradeoff: slow; re-reads the same files repeatedly; no efficiency gain from shared context.

**Approach B: Category-first (script-grouped) analysis**
Group the 10 beads by which script they reference, read each script once while analyzing all relevant beads for that script at once. Scripts: build-review-prompts.sh (0t31, 2585), parse-progress-log.sh (456u, 9p4i, dsaa, o7ji, w6lr), install-hooks.sh (71ye, 5365 partial), scrub-pii.sh (w2gj, 5365 partial).

Tradeoff: efficient for shared context; slightly harder to track per-bead evidence chains. Selected this approach.

**Approach C: Duplicate-first filtering**
Identify all known duplicate pairs first (456u/w6lr, dsaa/o7ji), reduce the effective bead count to 8 unique issues, then deeply analyze only the unique ones. Assign DUPLICATE_SUSPECT to the redundant beads and STILL_VALID/ALREADY_FIXED to the canonical ones.

Tradeoff: clean separation between duplication analysis and content analysis, but means treating duplicate pair members differently rather than applying a consistent schema. Loses opportunity to confirm both members point at the same location.

**Approach D: Evidence-matrix approach**
Before assigning any verdict, build a structured matrix of (bead_id, claimed_location, current_code_state, verdict_trigger) for all 10 beads. Then fill in verdicts from the matrix rather than deciding on-the-fly.

Tradeoff: forces explicit documentation before committing to verdicts, reduces confirmation bias; adds one extra pass. Could be combined with B.

## 2. Selected Approach

**Approach B (Category-first) with Approach D (evidence-matrix discipline)**

Read each of the 4 scripts once. While reading, evaluate all beads that reference that script simultaneously. For each bead, document the specific file/line evidence before assigning a verdict — applying the evidence-matrix rigor from Approach D without a separate pre-pass.

Rationale: The 10 beads span only 4 scripts. Reading each script once while evaluating all associated beads is strictly more efficient than per-bead sequential reads. The evidence-matrix discipline ensures ALREADY_FIXED verdicts cite specific line numbers, not general impressions. For the known duplicate pairs, both members are marked DUPLICATE_SUSPECT with cross-references, satisfying acceptance criterion 4.
