## Root Cause Analysis

**Design issue**: The preview is a **template** that gets combined with task-specific context (task-7xvw.md) by the orchestrator. The preview contains the universal 6-step structure, while context-specific details (file paths, root cause, acceptance criteria) live in the task context markdown.

This design is **reasonable for reuse** but creates an **audit problem**: When Pest Control receives the preview in isolation, it sees incomplete information. The preview lacks:
1. Real file paths (delegated to task context)
2. Root cause explanation (delegated to task context)
3. Line number specificity (delegated to task context via "full-file" marking in context, not in preview)

**The two-file design works if agents receive BOTH files**, but the preview alone fails the CCO checklist because the checklist expects a **self-contained prompt**.
