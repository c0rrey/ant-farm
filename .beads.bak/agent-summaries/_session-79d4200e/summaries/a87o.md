## 1. Approaches Considered

**Approach A — Replace per-task with session-wide only (simplest)**
- Remove the per-task CCO pattern entirely from both L27-28 (overview table) and the write instruction.
- Point only to `pc-session-cco-impl-{timestamp}.md`.
- Tradeoff: Loses documentation of the rare per-task serial audit case, which could confuse future implementers if they need to audit a single Dirt Pusher prompt in isolation.

**Approach B — Keep per-task as primary, add session-wide as alternate**
- Leave L27-28 unchanged; add a secondary session-wide option in the write instruction.
- Tradeoff: Perpetuates a misleading "primary" framing for the per-task format, which is the rare case in practice. Misrepresents actual usage.

**Approach C — Update the write instruction only, leave overview table unchanged**
- Update only the CCO write instruction (old L178) to document both batch and per-task modes.
- Leave the overview table (L26-33) as-is.
- Tradeoff: The overview table still categorizes CCO as "task-specific," creating a contradiction between the table and the updated write instruction. Partial fix.

**Approach D — Update both the overview table (L26-33) and the write instruction, with session-wide as the primary pattern (selected)**
- The root cause is documented in two places: the overview naming conventions table and the per-task write instruction.
- Move CCO from the task-specific row to the session-wide row in the overview table.
- Update the write instruction to document batch mode as primary (with `pc-session-cco-impl-{timestamp}.md`) and per-task as the fallback for isolated audits.
- Tradeoff: More edits, but produces an internally consistent document that accurately reflects practice at both the overview and the instruction level.

## 2. Selected Approach

**Approach D** — Update both the overview table and the write instruction.

Rationale: The bug exists in two documentation layers. Fixing only the write instruction (Approach C) leaves the overview table contradicting it. The overview table is read first by Pest Control during orientation; if it says CCO is "task-specific," that framing shapes how the write instruction is interpreted. Both layers must agree. Session-wide batch is the actual primary pattern in all observed sessions; per-task should be labeled as the rare isolated case.
