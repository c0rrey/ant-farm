### F-005
**File**: `orchestration/RULES.md:L146`
**Severity**: P3
**Category**: Inconsistent formatting / style drift
**Description**: The timestamp format comment reads `(YYYYMMDD-HHMMSS)` with uppercase `HHMMSS`. However, the bash example on L151 uses `date +%Y%m%d-%H%M%S` (lowercase `%H%M%S` which is standard strftime). The text format hint and the actual shell format string use different capitalization conventions — `HHMMSS` (uppercase) vs `HHmmss` (mixed case as used in `checkpoints.md:L42` and `GLOSSARY.md:L51`). A reader copying the format pattern may use the wrong casing.
**Suggested fix**: Normalise to `YYYYMMDD-HHmmss` throughout to match the canonical definition in `checkpoints.md:L42`.

## Preliminary Groupings

### Group A: Dead/Broken References (F-001, F-002)
Both findings in `SETUP.md` reference the same non-existent file (`docs/installation-guide.md`). Root cause: a file was planned or mentioned but never created, and no one updated the references when the decision was made not to create it. Fix: one atomic change to SETUP.md.

### Group B: CLAUDE.md Sync Divergence (F-003, F-004)
The repo `CLAUDE.md` and its synced `~/.claude/CLAUDE.md` copy have diverged in step content and numbering. Root cause: the sync hook copies the file on push, but both copies were apparently edited independently. F-003 (stale step reference parenthetical) and F-004 (extra steps missing from synced copy) are symptoms of the same underlying drift.

### Group C: Timestamp Format Inconsistency (F-005)
One-off capitalization inconsistency in RULES.md for the timestamp format string. Standalone finding — not part of a broader pattern.

### Group D: Structural/Table Issues (F-006, F-010)
F-006 (incomplete Hard Gates table row) and F-010 (strikethrough deprecation pattern in tables) are both table formatting issues that reduce scannability. Different tables, same underlying cause: no consistent table style guide for deprecated/special entries.

### Group E: Stale Snapshot Content (F-008, F-009)
F-008 (static audit table in PLACEHOLDER_CONVENTIONS.md) and F-009 (buried sunset clause) are both cases where content that was accurate at time of writing will silently become stale. Same root cause: no convention for marking point-in-time content vs. living reference content.

### Group F: Cross-Reference Precision (F-011)
Single finding about an underspecified cross-reference. Standalone.
