### Finding 10: implementation.md uses `<angle-bracket>` placeholders inconsistently with the Tiered Placeholder Convention

- **File(s)**: `orchestration/templates/implementation.md:9-120`
- **Severity**: P3
- **Category**: consistency
- **Description**: `implementation.md` uses `<angle-bracket>` placeholder syntax (`<task-type>`, `<task-id>`, `<session-dir>`, `<file>`, etc.) throughout the agent prompt template and summary doc template. Readers of PLACEHOLDER_CONVENTIONS.md see no explanation for why `implementation.md` is allowed to use a completely different syntax.
- **Suggested fix**: Add a brief paragraph to PLACEHOLDER_CONVENTIONS.md under "Exceptions and Special Cases" formally describing angle-bracket placeholders as "human-readable placeholders in instruction documents that the agent fills in from context (no Queen substitution required)." The Pantry author note at line 110 acknowledges this but never defines it as a rule.
