### DRIFT-001 — P1: DECOMPOSE_DIR path mismatch between skills/plan.md and orchestration/RULES-decompose.md

**File**: `skills/plan.md:120` vs `orchestration/RULES-decompose.md:118`

**Severity**: P1

**Category**: Value propagation — path constant changed in one location but stale in another

**Description**: `skills/plan.md` sets:
```
DECOMPOSE_DIR=".crumbs/sessions/_decompose-${DECOMPOSE_ID}"
```
`orchestration/RULES-decompose.md` sets:
```
DECOMPOSE_DIR=".beads/decompose/_decompose-${DECOMPOSE_ID}"
```

`skills/plan.md` is the entry point — it creates `DECOMPOSE_DIR` under `.crumbs/sessions/` — and then routes to `RULES-decompose.md` (via Step 4). But `RULES-decompose.md` overwrites `DECOMPOSE_DIR` with a `.beads/decompose/` path at its own Step 0. The Planner ends up operating under the `.beads/` path while the skill's error messages and directory structure documentation describe `.crumbs/sessions/`. This is a runtime failure: the `research/` subdirectory is not created under `.crumbs/sessions/` (it is created under `.beads/decompose/`), and verification checks look in the wrong place.

**Additional affected lines**: `RULES-decompose.md:416` (directory tree diagram) and `RULES-decompose.md:427` still document `.beads/decompose/`.

**Suggested fix**: Align `RULES-decompose.md:118` to use `.crumbs/sessions/_decompose-${DECOMPOSE_ID}` and update the directory tree at line 416 to match. The `.crumbs/sessions/` path is canonical for the crumbs-backed system (per the design spec at `docs/superpowers/specs/2026-03-12-crumbs-and-decomposition-design.md:230`).

### DRIFT-004 — P2: skills/plan.md routes to RULES-decompose.md but does not create research/ subdirectory

**File**: `skills/plan.md:119-121` vs `orchestration/RULES-decompose.md:119`

**Severity**: P2

**Category**: Caller/consumer update — skill creates DECOMPOSE_DIR but omits required subdirectory that RULES-decompose.md expects

**Description**: `skills/plan.md:121` runs `mkdir -p "${DECOMPOSE_DIR}"` but does NOT create the `research/` subdirectory. `RULES-decompose.md:119` creates `mkdir -p "${DECOMPOSE_DIR}/research"` — but only for its own (stale) `.beads/decompose/` path. If DRIFT-001 is fixed so both files agree on `.crumbs/sessions/`, then after the fix: `skills/plan.md` creates the directory root and `RULES-decompose.md` creates `research/`. The correct fix is for `skills/plan.md` to own the full directory scaffolding (`mkdir -p "${DECOMPOSE_DIR}/research"`) and for `RULES-decompose.md` to skip its own `mkdir` when already initialized.

**Suggested fix**: Align the mkdir in `skills/plan.md:121` to `mkdir -p "${DECOMPOSE_DIR}/research"` so the subdirectory is available before any agent writes. Alternatively, document clearly which layer is responsible for each path.

## Preliminary Groupings

### Root cause A: bd → crumb CLI migration incomplete in Architect-facing templates

DRIFT-006 (P1) and DRIFT-007 (P3) are the primary findings from the correctness reviewer's tip. The `bd` → `crumb` migration updated the Queen's orchestration layer but left the entire Architect executable workflow (`decomposition.md` Step 7, `architect-skeleton.md` Step 7 snippet, `RULES-decompose.md` prohibition lists) using `bd` commands. The Architect agent reads and runs these commands directly at runtime — this is not documentation drift, it is a broken tool chain.

### Root cause B: Path base directory migration (.beads/ → .crumbs/) not fully propagated

Findings DRIFT-001, DRIFT-002, DRIFT-003, DRIFT-004 share this root cause: the system migrated from `.beads/decompose/` to `.crumbs/sessions/`, but `RULES-decompose.md` and all template/agent example strings were not updated.

- DRIFT-001 (P1) is the functional breakage — RULES-decompose.md overwrites the correct path.
- DRIFT-003 (P2) is the documentation leak — agents boot with the wrong example.
- DRIFT-002 (P2) is the vocabulary leak — forager-skeleton.md still uses beads-era terms.
- DRIFT-004 (P2) is the structural consequence — mkdir doesn't create research/ in the right place.

Root causes A and B are related: both stem from the same incomplete bd→crumb migration. A single fix commit addressing all of `RULES-decompose.md`, `decomposition.md`, and `architect-skeleton.md` would resolve both root causes simultaneously.

### Root cause C: setup.sh / init skill responsibility boundary not documented

Finding DRIFT-005 stands alone. It's a documentation gap rather than a functional failure.
