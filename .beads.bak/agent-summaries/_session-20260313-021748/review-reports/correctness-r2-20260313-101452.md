- [x] Unsubstituted `{CODEBASE_ROOT}` produces an explicit error instead of silent misclassification — guard at line 129 checks `[ -d "${CODEBASE_ROOT}" ]`; if the placeholder is not substituted the check fails with a visible error message

  **Note on `find` quoting change**: The original code used `find "{CODEBASE_ROOT}"` (literal curly-brace string, no `$`). The fix updates to `find "${CODEBASE_ROOT}"` (shell variable expansion). The accompanying prose note says "substitute `{CODEBASE_ROOT}` with the absolute repo root path before running." LLM agents that follow these instructions typically set `CODEBASE_ROOT` as a shell variable first, so `${CODEBASE_ROOT}` is the correct expansion form. The old quoted-literal form was also usable (agents would substitute the literal string). Both forms work; the new form is more conventional shell and consistent with the guard expression.
