## Check 3b: Root Cause Spot-Check

Selected RC-R2-1 (P2, highest severity) for spot-check.

**RC-R2-1** claims `skills/plan.md:140` contains `printf '%s\n' '<INPUT_TEXT>'` which creates a single-quote injection vulnerability.

Actual code at line 140:
```bash
printf '%s\n' '<INPUT_TEXT>' > "${DECOMPOSE_DIR}/input.txt"
```
CONFIRMED — `<INPUT_TEXT>` is wrapped in literal single quotes. A spec containing an apostrophe (e.g., "It's a requirement") would break the shell command with a syntax error.

RC-R2-1 also claims `skills/plan.md:131` uses `--argjson class_score <CLASS_SCORE>`.

Actual code at line 131:
```bash
  --argjson class_score <CLASS_SCORE> \
```
CONFIRMED — `--argjson` treats the value as raw JSON; a non-integer substitution causes `jq` to exit non-zero and `manifest.json` goes unwritten.

Root cause explanation is accurate. Fix direction (use shell variable `${INPUT_TEXT}`, switch `--argjson` to `--arg` for class_score) is correct.

**RC-R2-1 spot-check: CONFIRMED**

## Check 4: Priority Calibration

- **RC-R2-1 (P2)** — JUSTIFIED. Single-quote injection via `printf '%s\n' '<INPUT_TEXT>'` breaks on any spec with an apostrophe. Real-world specs routinely contain contractions; this is a higher-probability failure than the original EC-05 heredoc collision it replaced. Silently corrupts the session with no user-visible error.
- **RC-R2-2 (P3)** — JUSTIFIED. Inconsistent guard on `backup_and_copy` at non-loop call sites; causes hard abort with no diagnostic. P3 because backup failure at those sites is rare and `set -e` does surface a line number.
- **RC-R2-3 (P3)** — JUSTIFIED. Tmpfile leak on signal kill is a common shell pattern limitation with minimal production impact.

**Severity conflict flag (not a CCB failure, but requires disclosure)**:

The Correctness reviewer assessed `skills/plan.md:140` and concluded PASS, calling `printf '%s\n' '<INPUT_TEXT>'` "correct instruction text for an LLM agent" and "not a regression." The Edge Cases reviewer assessed the same line and filed R2-EC-01 as P2, identifying the single-quote injection path. This is a 2-level discrepancy (PASS vs P2) on the same surface in the same file.

CCB resolution: **Edge Cases assessment prevails.** The Correctness reviewer's framing — "this is a pre-existing constraint of LLM-executed bash instructions" — does not hold. The R1 finding (EC-05) was specifically about heredoc delimiter collision in `input.txt` construction. The fix changed the construction method from heredoc to `printf`. The question is whether the new method introduces a new failure mode. It does: any spec with a single quote character breaks the shell command. This is not a "pre-existing constraint" — it is a new failure mode introduced by the fix. The Correctness reviewer's acceptance criteria did not include a test for single-quote content; the Edge Cases reviewer identified the gap. R2-EC-01 is a valid P2 finding.

This discrepancy does not block the CCB verdict but should be documented in the consolidated report's Severity Conflicts section.

**Result: PASS** (with flagged severity discrepancy)
