### RC-4 Spot-Check: Shell operator-precedence bug in init skill

RC-4 claims `skills/init.md:40` contains `[ -f pyproject.toml ] || [ -f setup.py ] || [ -f requirements.txt ] && echo "python"` with incorrect operator precedence.

Actual code at line 40:
```bash
[ -f pyproject.toml ] || [ -f setup.py ] || [ -f requirements.txt ] && echo "python"
```
CONFIRMED — this is exactly the buggy expression. Shell `&&` binds tighter than `||`, so only `requirements.txt` presence triggers the echo. Python projects without `requirements.txt` (using only `pyproject.toml` or `setup.py`) will not be detected.

Actual code at line 43:
```bash
[ -f pom.xml ] || [ -f build.gradle ] && echo "java"
```
CONFIRMED — same pattern for Java: `pom.xml`-only projects not detected.

Fix direction (wrap in `{ ... }` grouping) correctly addresses the precedence issue.

**RC-4 spot-check: CONFIRMED**
