## Approaches

Four approaches were considered:

1. **Mirror existing guard pattern (selected):** Add `[ -f .crumbs/config.json ]` to the existing bash guard, matching the pattern already used in `init.md`, `plan.md`, and `status.md`. Minimal change, consistent with codebase conventions, no new concepts introduced.

2. **Delegate to `crumb doctor`:** Replace the manual file-existence checks with a call to `crumb doctor`, which presumably validates full initialization state. Rejected because `crumb doctor` output is parsed via grep elsewhere in the file (checks 1–2), making it heavier than needed for a simple guard; also, the error reference table at the bottom of the file already handles the "crumb CLI not installed" case separately, suggesting the guard should stay as a pure shell check.

3. **Add a dedicated `crumb status` / `crumb init check` command:** Introduce a new crumb CLI subcommand that validates both files and returns a machine-readable result. Rejected because it requires changes outside `skills/work.md` (CLI implementation), which is out of scope for this bug fix.

4. **Soft warning instead of hard stop:** Detect missing `config.json` and surface a warning (like the stale in_progress check) rather than a hard stop, allowing the user to proceed anyway. Rejected because a missing `config.json` causes immediate downstream failures with no recovery path mid-session — a hard stop is the correct UX.

## Files Changed

- `skills/work.md` — updated initialization guard at line 25

## Implementation

Changed the bash guard from:

```bash
[ -f .crumbs/tasks.jsonl ] || echo "NOT_INITIALIZED"
```

To:

```bash
[ -f .crumbs/tasks.jsonl ] && [ -f .crumbs/config.json ] || echo "NOT_INITIALIZED"
```

Also updated the prose below the guard from "If `.crumbs/tasks.jsonl` does not exist" to "If `.crumbs/tasks.jsonl` or `.crumbs/config.json` does not exist" to match the new behavior.

## Correctness Notes

Re-read: yes

The fix correctly mirrors the pattern from `plan.md` and `status.md`. A project with `tasks.jsonl` present but `config.json` missing will now be flagged as NOT_INITIALIZED before entering the Queen pipeline, preventing unclear downstream failures from `crumb` CLI calls that depend on `config.json`.
