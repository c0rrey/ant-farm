## 1. Approaches Considered

**Approach A: Pure bash, cp-based with explicit per-file backup loop (selected)**
Walk each source file individually, use `cmp -s` to detect changes, backup only when content differs, copy to destination. Single timestamp `TS` for entire run prevents duplicate backups on re-run.
- Pros: no external dependencies beyond coreutils; explicit control over every file; easy to audit; consistent with existing scripts/ conventions
- Cons: more verbose than rsync; uses `find` + process substitution for directory traversal

**Approach B: Extend sync-to-claude.sh with rsync (reject)**
Copy the existing script and add crumb.py installation + PATH check on top. rsync's `--archive` mode doesn't produce per-file timestamped backups — only CLAUDE.md was backed up in the original. Meeting the "backup all files" criterion would require wrapping rsync with per-file pre-copy checks, eliminating rsync's conciseness advantage.
- Reject: rsync not well-suited to the per-file backup requirement without significant wrapper complexity

**Approach C: Python script (reject)**
Write in Python using `shutil.copy2`, `pathlib.Path`, `filecmp.cmp`. Richer stdlib, typed, easier to test.
- Reject: all existing scripts in scripts/ are bash; Python adds a runtime dependency perception; the task is a shell installer that runs before the Python tool is even installed — a chicken-and-egg problem for crumb.py installation

**Approach D: Makefile install target (reject)**
Add `Makefile` with an `install` target to the repo root.
- Reject: project has no existing Makefile; adding one for a single task is over-engineering; diverges from the scripts/ shell script convention

## 2. Selected Approach with Rationale

Approach A — pure bash, cp-based with explicit per-file backup loop.

Rationale:
- Follows existing `set -euo pipefail` + `REPO_ROOT` + `[ant-farm]` log prefix conventions from sync-to-claude.sh
- Satisfies all acceptance criteria without external tool dependencies
- `backup_and_copy` helper is reusable and testable by inspection
- Single `TS` variable set at script start ensures one backup per file per invocation (idempotency criterion)
- `--dry-run` flag added as a bonus for safe inspection before first run
