## Preliminary Groupings

### Root Cause A: SESSION_DIR path not updated after crumbs migration (F1, F2, F3, F4, F5, F6)
The session artifact directory moved from `.crumbs/agent-summaries/_session-*` to `.crumbs/sessions/_session-*` during the crumbs migration. `RULES.md` and `skills/work.md` were updated correctly, but six scoped files still contain the old path — four template files (dependency-analysis.md, dirt-pusher-skeleton.md, scout.md, scribe-skeleton.md) and two instruction files (CLAUDE.md, AGENTS.md). These are all copies of the same stale value.

**Impact**: Agents constructing SESSION_DIR from a stale example will write artifacts to a path the workflow does not read from, causing silent data loss.

### Root Cause B: Procedure inconsistency between AGENTS.md and RULES.md (F7)
The landing workflow is defined in two places with slightly different step lists. The `crumb sync` step appears in one but not the other. This is low-severity because RULES.md is the authoritative reference, but the inconsistency will cause confusion when a developer cross-references the two.

### Root Cause C: setup.sh replacement of sync-to-claude.sh not propagated to docs (F8 partial, F9)
Documentation files reference the old script name. The pre-commit hook filename discrepancy (F8) may also be part of the same migration wave — `issues.jsonl` is the beads-era filename and `tasks.jsonl` is the crumbs-era filename.
