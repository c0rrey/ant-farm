**File**: `AGENTS.md:L3`
**Severity**: P2
**Category**: Naming / Misleading comment
**Description**: The file says "This project uses **crumb** for issue tracking" but throughout the broader codebase, the tool is also called `bd` (beads). ## Preliminary Groupings

- F-01 (AGENTS.md:L3) — `crumb` vs `bd` distinction unclear

- F-08 (RULES-decompose.md:L185: `surveyor-skeleton.md`) — may not exist

**Common cause**: Reader comments and template references not updated when scripts or files were renamed.

- F-07 (L110-115) — hook output message unverified

**Common cause**: Installation guide was written before `setup.sh` replaced `sync-to-claude.sh`, and the sync behavior description was not updated to reflect the new script's approach.

**Common cause**: Incremental deprecation without corresponding cleanup or clear "stop here" markers.

- F-18 (L149-150) — `FOCUS_AREAS_FILE` path derivation not explained
- F-19 (L222) — sed regex converts placeholder format, no comment

**Common cause**: Script was written with non-obvious path assumptions and transformation logic, without inline documentation explaining the design choices.

- F-12 (implementation.md: checkmark/cross emoji throughout info diet section)

**Common cause**: One template file uses emoji for visual emphasis while all others use plain text.

## Overall Assessment

**Score**: 7/10
**Verdict**: PASS WITH ISSUES

The documentation and templates are generally well-structured, with clear reader annotations, thorough placeholder conventions, and solid step-by-step instructions. The system is complex and the overall clarity is high for an orchestration framework of this scale.

The main clarity concerns are:
1. The `crumb` vs `bd` CLI ambiguity is a meaningful onboarding obstacle (F-01, F-09) — a new adopter cannot tell from any single document which CLI tool to use.
2. The installation guide contains a factually wrong claim about `rsync --delete` (F-06) that could mislead users about file deletion behavior.
3. The deprecated `pantry-review` entry in README (F-17) creates a misleading picture of what agents are active.
4. `build-review-prompts.sh` has a fragile implicit path assumption (F-18) that would be hard to diagnose on failure.

All other findings are P3 polish that would improve readability without blocking any workflow.
