## Approaches Considered

### 1. Direct string substitution (literal find-and-replace per occurrence)
**Strategy**: Enumerate every unique `bd <subcommand>` pattern in both files and replace each using targeted Edit calls, one pattern at a time.
**Pros**: Surgical precision; clear audit trail per change; no risk of false positive replacements.
**Cons**: Requires manually cataloguing all occurrences first; more individual Edit operations.
**Tradeoff**: Precision over automation speed.

### 2. Global word-boundary `bd` substitution in one pass
**Strategy**: Replace every word-boundary match of `bd` with `crumb` across both files in a single automated sweep, then replace `.beads/` with `.crumbs/`.
**Pros**: Comprehensive; cannot miss any `bd` occurrence.
**Cons**: Risk of false positives on `bd` appearing in non-command contexts (prose abbreviations, variable names, identifiers).
**Tradeoff**: Coverage over precision.

### 3. Pattern-by-pattern substitution using targeted Edit calls (selected)
**Strategy**: Grep both files to enumerate all `\bbd\b` occurrences with line numbers, then apply Edit calls scoped to each distinct pattern block.
**Pros**: Human-readable, auditable; handles multi-line context cleanly; avoids regex false positives; verifiable by re-grep.
**Cons**: More verbose than a scripted substitution; requires careful enumeration.
**Tradeoff**: Auditability and safety over automation speed.

### 4. Full file rewrite via Write tool
**Strategy**: Read each file entirely, perform all substitutions in memory, then overwrite the file using the Write tool in one atomic operation.
**Pros**: Atomic; handles all patterns simultaneously; single operation per file.
**Cons**: Produces an unreadable diff (entire file replaced); higher risk of inadvertent whitespace or formatting changes.
**Tradeoff**: Atomicity over diff clarity.

## Selected Approach
**Choice**: Approach 3 — pattern-by-pattern Edit calls after grep enumeration.
**Rationale**: This task is documentation editing where auditability matters. Edit calls produce granular diffs and are easier to verify. Unlike Approach 2, context-aware edits eliminated false-positive risk (e.g., the word "beads" as terminology was correctly left alone). Unlike Approach 4, individual edits preserve formatting and produce a clean, reviewable diff. Approach 1 was rejected because it lacked the initial grep enumeration step; without confirming all occurrences first, silent misses are likely.
