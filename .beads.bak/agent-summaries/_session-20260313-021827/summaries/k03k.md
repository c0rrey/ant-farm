**Approach A: Targeted line-by-line edits (Edit tool)**
Replace each `bd` reference individually using the Edit tool with enough surrounding context to ensure uniqueness. Most precise; zero risk of accidental changes to nearby content. Highest number of tool calls.

**Approach B: Full file rewrite (Write tool)**
Read each file in full, perform all substitutions in memory, write the complete file back. Fewer tool calls but introduces risk of subtly altering formatting or accidentally touching content that was not in scope.

Use `sed -i` with word-boundary regex (`\bbd\b`) to batch-replace across both files in one pass. Fast, but requires careful regex crafting to avoid matching `bd` inside longer tokens (e.g., `abd`, `build`), and offers less visibility into exactly what changed.

**Approach D: Edit tool with multi-line context blocks**
Group changes that are near each other into a single Edit call covering the surrounding block. Reduces total tool calls compared to Approach A while maintaining the same precision, since each block uniquely identifies a contiguous region.
