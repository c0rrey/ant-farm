# Task Summary: ant-farm-n56q — Migrate reviews.md (semantic)

## 1. Approaches Considered

**Approach 1: Regex-based global find-replace**
Use automated regex patterns to substitute all `\bbd\b` occurrences in a single pass. Fast and comprehensive, but risky: regex cannot distinguish between `bd create` in a bash block (requiring structural JSON conversion) and `bd list` in prose (requiring a flag-syntax change). The `bd label add` removal plus `review_source` injection is impossible to express as a simple substitution pattern.

**Approach 2: Line-by-line manual transformation**
Read every flagged line individually and apply a context-specific transform. Maximum precision, but extremely slow given 30+ references spread across 1157 lines in a file too large to read in one pass.

**Approach 3: Section-by-section targeted edits (SELECTED)**
Group the 30+ references by the 8 distinct command patterns documented in the task description (create, list, search, show, dep add, epic create, label add, update), read only the relevant surrounding context, and apply precise Edit tool calls per group. Balances semantic accuracy with implementation efficiency.

**Approach 4: Full file rewrite**
Load the entire file into context and rewrite it from scratch, ensuring zero `bd` references survive. Highest completeness guarantee but risks inadvertent changes to non-command text (Nitpicker workflow logic, Big Head dedup protocol, report format definitions). The file is also too large for a single Read pass, making this approach impractical without chunking that defeats its key advantage.

## 2. Selected Approach and Rationale

Selected **Approach 3** (section-by-section targeted edits). The 8 distinct command patterns each require a different semantic transformation — some are flag-syntax changes (`bd show` → `crumb show`), some require structural JSON conversion (`bd create --type=bug` → `crumb create --from-json {...}`), and one requires deletion plus field injection (`bd label add` → `review_source` in `--from-json`). Grouping by pattern allows applying the exact right transformation to each group while leaving all surrounding review logic untouched.
