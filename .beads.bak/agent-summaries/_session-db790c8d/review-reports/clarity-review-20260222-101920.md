### F-003
- **File:line**: `CONTRIBUTING.md:L147`
- **Severity**: P3
- **Category**: Documentation / misleading example
- **Description**: The `fill-review-slots.sh` test example passes changed files as `"file1.py\nfile2.py"` inside a double-quoted bash string. In bash, `\n` inside double quotes is a literal two-character sequence `\n`, not a newline. A contributor copying this example verbatim would pass a single malformed string `file1.py\nfile2.py` to the script rather than two newline-separated filenames. The example does not match the format the script likely expects.
- **Suggested fix**: Use `$'...'` quoting for literal newlines, or use a variable:
  ```bash
  ./scripts/fill-review-slots.sh <SESSION_DIR> \
    "abc1234..HEAD" $'file1.py\nfile2.py' "task-1 task-2" \
    "$(date +%Y%m%d-%H%M%S)" 1
  ```
  Or, if the script accepts space-separated files: update the comment to document the expected delimiter.
