## Affected Files

## Root Cause
Several functions call `open()` or `path.touch()` without catching `OSError`, causing raw Python tracebacks instead of clean `die()` messages when files are unreadable, missing (TOCTOU race), or on a read-only filesystem. The codebase already has the correct pattern in `read_config()` at L139.

## Expected Behavior
All file open/touch operations should be wrapped in `try/except OSError` with clean `die()` messages, following the pattern in `read_config()`.

## Acceptance Criteria
1. `read_tasks()` produces a clean `die()` error message on permission error or missing file (TOCTOU race)
2. `FileLock.__enter__` produces a clean `die()` error message on read-only filesystem or permission error
3. `iter_jsonl()` produces a clean `die()` error message on file access failure
