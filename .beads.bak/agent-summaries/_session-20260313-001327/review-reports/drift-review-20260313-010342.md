## Findings Catalog

### DRIFT-1
**File**: `/Users/correy/projects/ant-farm/crumb.py:520`
**Severity**: P2
**Category**: Value propagation / Internal API drift

**Description**: `cmd_list --parent` filter checks only the top-level `parent` field (`t.get("parent")`), but `crumb link --parent` stores the parent link in `links.parent` (line 1030). Crumbs whose parent is set via `crumb link --parent <id>` will never appear in `crumb list --parent <id>` results. The rest of the codebase (`_get_trail_children` line 401, `_auto_close_trail_if_complete` line 428, `cmd_doctor` lines 1759-1762) consistently reads from `links.parent`.

**Impact**: Any caller using `crumb list --parent AF-T1` to enumerate a trail's crumbs will get silently incomplete results if those crumbs were linked via `crumb link` rather than created with an explicit top-level `parent` field.

**Suggested fix**: Change line 520 to check both locations, matching the pattern used in `_get_trail_children`:
```python
if args.parent:
    results = [
        t for t in results
        if t.get("parent") == args.parent
        or (isinstance(t.get("links"), dict) and t.get("links", {}).get("parent") == args.parent)
    ]
```
